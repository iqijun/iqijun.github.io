    /*devServer:{
        historyApiFallback:true,

        inline:true,

    resolve: {

1.名词对比
sql术语 mongo术语 解释/说明
database database 数据库
table collection 表/集合
row document 记录/文档
column field 数据字段/域
index index 索引
primary key primary key 主键,MongoDB自动将_id字段设置为主键
2.数据库操作
  1. 查看数据库 show dbs
  2. 显示当前数据库 db
  3. 切换数据库 use ${dbName} 如果不存在名字为{dbname}的数据库则直接创建
  4. mongo保留数据库：
    1. admin： 从权限的角度来看，这是"root"数据库。要是将一个用户添加到这个数据库，这个用户自动继承所有数据库的权限。一些特定的服务器端命令也只能从这个数据库运行，比如列出所有的数据库或者关闭服务器。
    2. local: 这个数据永远不会被复制，可以用来存储限于本地单台服务器的任意集合
    3. config: 当Mongo用于分片设置时，config数据库在内部使用，用于保存分片的相关信息。
  5. 删除数据库
    1. db.dropDatabase(); 删除当前数据库
3.集合操作
  1. 删除集合：db.collection.drop()
4.文档操作
1. 插入文档 ：db.collection.insert({name:"cruise"});
  1. db.col.save(document) 命令。如果不指定 _id 字段 save() 方法类似于 insert() 方法。如果指定 _id 字段，则会更新该 _id 的数据。
  2. 更新操作
    1. db.collection.update( <query>, <update>, { upsert: <boolean>, multi: <boolean>, writeConcern: <document> } )
query : update的查询条件，类似sql update查询内where后面的。
update : update的对象和一些更新的操作符（如$,$inc...）等，也可以理解为sql update查询内set后面的
upsert : 可选，这个参数的意思是，如果不存在update的记录，是否插入objNew,true为插入，默认是false，不插入。
multi : 可选，mongodb 默认是false,只更新找到的第一条记录，如果这个参数为true,就把按条件查出来多条记录全部更新。
writeConcern :可选，抛出异常的级别。
4.移除操作
db.collection.remove( <query>, { justOne: <boolean>, writeConcern: <document> } )
query :（可选）删除的文档的条件。
  • justOne : （可选）如果设为 true 或 1，则只删除一个文档。
  • writeConcern :（可选）抛出异常的级别。
5.查询操作
and条件查询：
MongoDB 的 find() 方法可以传入多个键(key)，每个键(key)以逗号隔开，及常规 SQL 的 AND 条件。
db.col.find({key1:value1, key2:value2}).pretty()
or条件查询：
db.col.find( { $or: [ {key1: value1}, {key2:value2} ] } ).pretty()
条件操作符：
  • (>) 大于 - $gt
  • (<) 小于 - $lt
  • (>=) 大于等于 - $gte
  • (<= ) 小于等于 - $lte
6.排序
在MongoDB中使用使用sort()方法对数据进行排序，sort()方法可以通过参数指定排序的字段，并使用 1 和 -1 来指定排序的方式，其中 1 为升序排列，而-1是用于降序排列。
db.COLLECTION_NAME.find().sort({KEY:1})
7.索引
db.COLLECTION_NAME.ensureIndex({KEY:1})
语法中 Key 值为你要创建的索引字段，1为指定按升序创建索引，如果你想按降序来创建索引指定为-1即可

  1. explain()
实例：
db.users.find({gender:"M"},{user_name:1,_id:0}).explain()

分析：
  • indexOnly: 字段为 true ，表示我们使用了索引。
  • cursor：因为这个查询使用了索引，MongoDB中索引存储在B树结构中，所以这是也使用了BtreeCursor类型的游标。如果没有使用索引，游标的类型是BasicCursor。这个键还会给出你所使用的索引的名称，你通过这个名称可以查看当前数据库下的system.indexes集合（系统自动创建，由于存储索引信息，这个稍微会提到）来得到索引的详细信息。
  • n：当前查询返回的文档数量。
  • nscanned/nscannedObjects：表明当前这次查询一共扫描了集合中多少个文档，我们的目的是，让这个数值和返回文档的数量越接近越好。
  • millis：当前查询所需时间，毫秒数。
  • indexBounds：当前查询具体使用的索引。
2.使用 hint()
虽然MongoDB查询优化器一般工作的很不错，但是也可以使用hints来强迫MongoDB使用一个指定的索引。
这种方法某些情形下会提升性能。 一个有索引的collection并且执行一个多字段的查询(一些字段已经索引了)。
如下查询实例指定了使用 gender 和 user_name 索引字段来查询：
db.users.find({gender:"M"},{user_name:1,_id:0}).hint({gender:1,user_name:1})
15.原子操作
mongodb不支持事务，所以，在你的项目中应用时，要注意这点。无论什么设计，都不要要求mongodb保证数据的完整性。
但是mongodb提供了许多原子操作，比如文档的保存，修改，删除等，都是原子操作。
所谓原子操作就是要么这个文档保存到Mongodb，要么没有保存到Mongodb，不会出现查询到的文档没有保存完整的情况。

16.ObjectId
1. 组成：
ObjectId 是一个12字节 BSON 类型数据，有以下格式：
  • 前4个字节表示时间戳
  • 接下来的3个字节是机器标识码
  • 紧接的两个字节由进程id组成（PID）
  • 最后三个字节是随机数。

我为什么写微信公众账号

Posted on 2016-12-02 Edited on 2019-02-20 In 杂谈 Word count in article: 977 Reading time ≈ 4 mins.
有些人25岁就“死了”，只是到75岁才“埋葬”

前几天在朋友圈发了张二维码，使我的小公众账号涨粉高达个位数，也是没shei了。但是有个朋友们十分关心的问题我还是挺想回答一下的，如题，我为啥要搞这么个公众账号。

其实早就申请了这么个公众账号了，应该是long long ago的2013年了吧。当时是一时兴起，写了个公众账号的程序，最终也没能实现什么吸引人的功能，一个简单的回复，一个查询天气预报，然后就不了了之了。。。直到最近看到各大头条报道公众账号盈利n千万，我的心深深被震撼了，在后台看了下我两位数的粉丝量，这新闻当然不是我开始写点东西的原因了。

好吧，言归正传。突然想写点东西，其实就是为了给自己找点乐子。不至于使生活太程序化。还记得那句话，有些人25岁就“死了”，只是到75岁才“埋葬”。最近几个月对引言的上半部分有点感触吧，这不是笑话，今年我二十五周岁！为什么感觉像“死了”呢？先郑重的以文字形式吐槽下今年的加班生活吧，一句话以蔽之，除了九月份双休，其他时间定期或不定期加班。为什么有的定期有的不定期呢，因为上一家公司加班不定期，现在这家定期，所以说程序猿，你还以为换工作能逃脱加班的命运吗。倒不是说拒绝加班，IT dog 们都感受过来自工作量，上线时间，项目进度，dead line 压力，不加班怎么去完成任务呢。但一遍遍的复制粘贴，一次次的重复，没有丝毫新知识，新鲜事物来刺激下大脑，把激情一点点的都耗光了。所以就像机器一样，今天重复昨天，明天重复着今天。是搬砖还是在创造？所以说呢，写点东西，算是给自己个小小的挑战，也给自己跳出一天天的重复找个突破口。

本来想把第一篇文章写的长一点，漂亮点，但写了这么两段话，突然感觉写不下去了。那么咱们扯点别的吧。

要不谈谈理想？

刚刚读完一本书《财务自由之路》，一本关于财富的书。书中讲到了如何赚到第一个100万，如何走向财富自由的三个阶段：财务保障，财富安全，财务自由。但在实现这些之前，你需要明确你真正想要什么，你的目标是什么？

暂停下，想想你还有目标吗？

我想了好久，最后得出的结论是：先攒够了买房的钱，其他的事情慢慢来。之所以其他的事情要慢慢来，是因为其他的事情想不清了。想不清还要去做哪些事。当时吹牛x,说下要一起做改变世界的程序员的“誓言”，现在却感觉离改变世界越走越远。经过三年的工作，发现同学们都开始重新定位和思考自己的职业之路，做出新的选择。想了想好像除了code code and code 啥都不会了，定位到哪依然是个想不清的问题。

为了不将负面情绪表达出来，就不再往下写了。

快速结束语：在想清楚要干什么之前，我的梦想依然是做个改变世界的程序员！
